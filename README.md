# hrod.dev - Blog de Infraestructura, Linux y Seguridad

¡Bienvenido! Este es un proyecto de código abierto dedicado a compartir conocimiento técnico sobre el ecosistema de **Linux, Infraestructura, Seguridad y Nube**. El objetivo principal es crear una base de conocimiento en español para entusiastas y profesionales que deseen aprender y crecer en estas áreas.

---

## 🚀 Áreas de Enfoque

Este espacio nace con la intención de democratizar el acceso a información técnica de calidad en idioma español. Nos centramos en:

*   **🐧 Linux:** Administración de sistemas, personalización (dots), kernel y optimización.
*   **🏗️ Infraestructura & DevOps:** Automatización, Infrastructure as Code (IaC), CI/CD y monitoreo.
*   **🛡️ Seguridad:** Ciberseguridad, hardening de servidores, criptografía y seguridad ofensiva/defensiva.
*   **☁️ Cloud Computing:** Arquitecturas en la nube, gestión de servicios y soluciones escalables.

## 💡 Filosofía y Código Abierto

Promovemos activamente el uso de **herramientas de código libre (FOSS)**. Creemos que el conocimiento debe ser libre y accesible para todos. Este proyecto no es solo un blog, es una comunidad en construcción donde se valora la curiosidad y el rigor técnico.

## 🤝 ¿Cómo puedes contribuir?

¡Cualquier persona puede contribuir! No importa tu nivel de experiencia, siempre hay algo en lo que puedes ayudar:

1.  **Contenido:** Escribe artículos o tutoriales sobre los temas mencionados arriba.
2.  **Correcciones:** ¿Encontraste un error ortográfico o un comando que no funciona? ¡Reportalo o corrígelo!
3.  **Mejoras Técnicas:** Sugiere mejoras en el diseño o funcionalidades del sitio.

### Pasos para colaborar:
1.  Haz un **Fork** de este repositorio.
2.  Crea una nueva rama (`git checkout -b feature/mi-aporte`).
3.  Realiza tus cambios y haz un commit descriptivo.
4.  Abre un **Pull Request** explicando detalladamente tu contribución.

## 🛠️ Arquitectura del Proyecto

Este blog está construido con tecnologías modernas y declarativas:
- **[Hugo](https://gohugo.io/):** El generador de sitios estáticos más rápido, ideal para documentación técnica.
- **Markdown:** Todo el contenido se gestiona de forma sencilla mediante archivos `.md`.
- **github actions** Automatización para un despliegue continuo.
- **github pages** Alojamiento del sitio.
